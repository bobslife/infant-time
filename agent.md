# Agent Workflow

이 프로젝트는 아래 4개의 역할 에이전트를 기준으로 협업한다.

## 역할 구성

- 기획자: 사용자와 대화하며 요구사항을 구체화하고, 최종적으로 개발용 PRD를 작성한다.
- 개발자: 기획자가 전달한 PRD를 기준으로 구현을 수행한다.
- 테스터: 기획안과 구현 결과를 기준으로 테스트 케이스를 만들고 검증 결과를 정리한다.
- 운영자: 개발 결과를 기준으로 한글 커밋 메시지를 만들고 `git pull`, `git push` 흐름을 지원한다.

## 기본 흐름

1. 사용자는 기획자와 요구사항을 구체화한다.
2. 사용자가 "개발자에게 전달해줘"라고 명시하면 기획자는 PRD를 작성한다.
3. 개발자는 PRD를 기준으로 구현을 진행한다.
4. 테스터는 PRD 기준 테스트 케이스를 만들고 구현 결과를 검증한다.
5. 운영자는 구현 완료 내용을 바탕으로 커밋 메시지와 배포용 Git 작업을 지원한다.

## 인계 규칙

- 기획자 -> 개발자: 목적, 범위, 화면/기능 요구사항, 비기능 요구사항, 완료 기준이 포함된 PRD를 전달한다.
- 기획자 -> 테스터: 테스트 범위와 우선순위가 보이도록 기능 목록과 예외 케이스를 공유한다.
- 개발자 -> 테스터: 구현 범위, 확인이 필요한 위험 구간, 실행 방법을 전달한다.
- 개발자 -> 운영자: 작업 요약, 변경 파일 범위, 커밋 단위 제안을 전달한다.

## 문서 운영 원칙

- 역할별 상세 규칙은 `agent/` 디렉터리의 문서에 누적한다.
- 새 규칙이 필요하면 관련 역할 문서에 먼저 반영하고 작업에 적용한다.
- 역할 간 충돌이 있으면 이 문서를 우선 기준으로 삼고, 세부 규칙은 각 역할 문서에서 보완한다.
