# Infant Time Account-Based MVP PRD

## 배경

신생아 돌봄 과정에서 보호자는 수유, 수면, 소변, 대변 기록을 자주 남겨야 한다. 부부가 번갈아 돌보는 상황에서는 마지막 수유 시점, 오늘의 기록 현황, 최근 기록을 빠르게 확인할 수 있어야 한다.

기존 로컬 저장 중심 MVP에서 실사용 가능한 계정 기반 MVP로 범위를 확장한다. 사용자는 이름, 이메일, 비밀번호로 계정을 만들고, 최초 가입 시 아기 이름과 생년월일을 등록한 뒤 기록 화면을 사용한다.

## 목표

- 이름, 이메일, 비밀번호 기반 회원가입/로그인으로 사용자를 식별한다.
- 최초 가입 후 아기 이름과 생년월일을 등록할 수 있게 한다.
- 회원가입 또는 로그인 후 세션을 유지해 다음 방문 시 추가 로그인 없이 진입할 수 있게 한다.
- MVP에서는 사용자당 아기 1명만 지원하되, 추후 다중 아기 지원이 가능하도록 데이터 모델을 설계한다.
- 아이폰에서 사용하기 좋은 모바일 우선 웹앱을 제공한다.
- 홈, 입력, 프로필 탭으로 주요 화면을 분리한다.
- 수유, 수면, 소변, 대변 기록을 빠르게 남길 수 있게 한다.
- 오늘 기준 기록 현황과 최근 기록을 보여준다.
- 마지막 수유 이후 4시간이 경과하면 자동 푸시 알림 대신 화면 내 경고 상태로 알려준다.
- Supabase Auth, Supabase Database, Vercel 배포를 전제로 구현한다.

## 대상 사용자

- 신생아를 돌보는 부모
- 아이폰으로 빠르게 기록하고 싶은 사용자
- MVP 단계에서는 이메일 계정 1개로 사용하는 단일 보호자 또는 부부 공유 계정 사용을 우선 가정한다.

## 핵심 사용자 흐름

1. 사용자는 모바일 브라우저에서 서비스에 접속한다.
2. 계정이 없는 사용자는 이름, 이메일, 비밀번호를 입력해 회원가입한다.
3. 계정이 있는 사용자는 이메일과 비밀번호로 로그인한다.
4. 최초 가입 사용자는 아기 이름과 생년월일을 등록한다.
5. 아기 프로필 등록이 완료되면 홈 화면으로 이동한다.
6. 이미 로그인 세션이 유지된 사용자는 추가 로그인 없이 홈 화면으로 진입한다.
7. 홈 화면에서 아기 이름, 오늘 요약, 마지막 수유, 마지막 대변, 최근 기록을 확인한다.
8. 사용자는 하단 `입력` 탭으로 이동한다.
9. 사용자는 수유, 수면, 소변, 대변 중 하나를 아이콘 버튼으로 선택한다.
10. 선택한 기록 타입에 맞는 입력값을 작성하고 저장한다.
11. 저장 완료 후 홈 또는 최근 기록 데이터가 즉시 갱신된다.
12. 마지막 수유로부터 4시간 이상 지나면 홈 화면에서 경고 상태를 표시한다.

## 화면 구성

### 1. 로그인/회원가입 화면

- 서비스 이름
- 로그인 폼
  - 이메일
  - 비밀번호
  - 로그인 버튼
- 회원가입 폼
  - 이름
  - 이메일
  - 비밀번호
  - 비밀번호 확인
  - 회원가입 버튼
- 로그인/회원가입 전환 UI
- 회원가입 완료 후 자동 로그인
- 로그인 세션이 남아 있으면 이 화면을 건너뛰고 앱으로 진입
- 로그인 실패 시 오류 메시지

### 2. 최초 아기 프로필 등록 화면

- 아기 이름 입력
- 아기 생년월일 입력
- 저장 버튼
- 저장 완료 후 홈 화면 진입

### 3. 공통 하단 탭

- 홈
- 입력
- 프로필

하단 탭은 모바일 화면 하단에 고정한다. 각 탭은 아이콘과 문구를 함께 표시한다.

### 4. 홈 화면

- 상단
  - 아기 이름을 주요 헤드라인으로 표시
  - 오늘 날짜 표시
- 상태 요약 영역
  - 마지막 수유
    - 기록 시각
    - `지금으로부터 n시간 n분 전` 형식의 상대 시간
  - 마지막 대변
  - 오늘 수유 횟수
  - 오늘 수면 횟수 또는 총 수면 시간
  - 오늘 소변 횟수
  - 오늘 대변 횟수
- 경고 상태
  - 마지막 수유 이후 4시간 이상 경과하면 홈 화면에서 강조 표시
  - 새 수유 기록이 등록되면 경고 해제
- 최근 기록
  - 섹션 문구는 `최근 기록`만 사용한다.
  - `최신순 목록` 문구는 사용하지 않는다.
  - 기록 타입, 시각, 핵심 상세값을 표시한다.

### 5. 입력 화면

- 상단에 기록 타입 선택 버튼 4개 표시
  - 수유
  - 수면
  - 소변
  - 대변
- 각 버튼은 아이콘과 문구를 함께 표시한다.
- 버튼 선택 시 아래 입력 영역이 해당 타입에 맞게 변경된다.
- 공통 입력값
  - 기록 시각
  - 메모 optional

#### 수유 입력

- 수유량 입력
- 기본값: 100ml
- 범위: 0ml ~ 300ml
- 단위: 5ml
- 좌우 스크롤 또는 슬라이더 방식으로 선택한다.
- 저장 후 최근 기록에는 예: `수유 100ml` 형태로 표시한다.

#### 수면 입력

- 수면 시작 시각
- 수면 종료 시각
- 종료 시각이 없으면 진행 중 수면 기록을 허용할지 여부는 구현 중 단순화를 위해 종료 시각 필수로 시작한다.
- 저장 후 최근 기록에는 수면 시간 요약을 표시한다.

#### 소변 입력

- 기록 시각
- 메모 optional
- 저장 후 최근 기록에는 `소변`과 기록 시각을 표시한다.

#### 대변 입력

- 대변 양
  - 적음
  - 보통
  - 많음
- 대변 색상
  - 황토색
  - 갈색
  - 진한 갈색
  - 쑥색
  - 다홍색
- 저장 후 최근 기록에는 예: `대변 보통 · 갈색` 형태로 표시한다.

### 6. 프로필 화면

- 아기 이름
- 생년월일
- 생후 정보
- 로그인 계정 정보
- 로그아웃 버튼

프로필 수정은 MVP에서 선택 사항이다. 단, 데이터 모델은 보호자 이름, 아기 이름, 생년월일 업데이트가 가능하도록 설계한다.

## 데이터 모델 초안

### profiles

- `id` uuid, primary key, Supabase auth user id 참조
- `name`
- `email`
- `created_at`
- `updated_at`

### babies

- `id` uuid, primary key
- `owner_id` uuid, profiles.id 참조
- `name`
- `birth_date`
- `created_at`
- `updated_at`

MVP에서는 사용자당 아기 1명만 사용한다. 추후 다중 아기 지원을 위해 `events`는 `baby_id`를 기준으로 연결한다.

### events

- `id` uuid, primary key
- `user_id` uuid, profiles.id 참조
- `baby_id` uuid, babies.id 참조
- `event_type` text
  - `feed`
  - `sleep`
  - `pee`
  - `poop`
- `occurred_at` timestamptz
- `ended_at` timestamptz nullable
- `amount_ml` integer nullable
- `poop_amount` text nullable
  - `small`
  - `normal`
  - `large`
- `poop_color` text nullable
  - `ocher`
  - `brown`
  - `dark_brown`
  - `green`
  - `red_orange`
- `note` text nullable
- `created_at`
- `updated_at`

## 기능 요구사항

### 인증

- 사용자는 이름, 이메일, 비밀번호로 회원가입할 수 있어야 한다.
- 회원가입 성공 후 자동으로 로그인되어야 한다.
- 사용자는 이메일과 비밀번호로 로그인할 수 있어야 한다.
- 로그인 세션이 남아 있으면 앱 재방문 시 추가 로그인 없이 진입해야 한다.
- 로그인하지 않은 사용자는 기록 화면에 접근할 수 없어야 한다.
- 사용자는 로그아웃할 수 있어야 한다.

### 아기 프로필

- 최초 가입 사용자는 아기 이름과 생년월일을 등록해야 한다.
- 아기 프로필이 없으면 홈/입력 화면 대신 프로필 등록 화면을 보여준다.
- MVP에서는 아기 1명만 지원한다.

### 기록

- 사용자는 수유, 수면, 소변, 대변 기록을 생성할 수 있어야 한다.
- 기록 생성 시 기본 시각은 현재 시각이어야 한다.
- 사용자는 기록 시각을 수동으로 수정할 수 있어야 한다.
- 수유 기록에는 수유량을 저장해야 한다.
- 수면 기록에는 시작 시각과 종료 시각을 저장해야 한다.
- 대변 기록에는 양과 색상을 저장해야 한다.

### 조회

- 사용자는 아기 이름을 홈 화면에서 볼 수 있어야 한다.
- 사용자는 마지막 수유 정보를 볼 수 있어야 한다.
- 마지막 수유 카드에는 `지금으로부터 n시간 n분 전` 형식의 상대 시간이 표시되어야 한다.
- 사용자는 마지막 대변 정보를 볼 수 있어야 한다.
- 사용자는 오늘의 수유, 수면, 소변, 대변 기록 현황을 볼 수 있어야 한다.
- 사용자는 최근 기록 목록을 최신순으로 볼 수 있어야 한다.

### 요약 및 분석

- 서비스는 마지막 수유 후 경과 시간을 분 또는 시간 단위로 표시해야 한다.
- 서비스는 오늘 기준 기록 수를 즉시 갱신해야 한다.
- 수면은 가능하면 오늘 총 수면 시간 또는 오늘 수면 횟수 중 하나를 요약으로 보여준다.

### 경고 상태

- 마지막 수유 이후 4시간 이상 경과하면 경고 상태를 표시해야 한다.
- 경고는 자동 푸시 알림이 아니라 화면 내 강조 UI로 제공한다.
- 새로운 수유 기록이 등록되면 경고 상태는 해제되어야 한다.

## 비기능 요구사항

- 모바일 우선 UI여야 한다.
- 아이폰 Safari에서 주요 기능이 정상 동작해야 한다.
- 하단 탭은 엄지 손가락 사용성을 고려해 충분한 터치 영역을 가져야 한다.
- 입력 화면은 타입별로 필요한 입력만 보여줘야 한다.
- 홈 화면은 최근 기록 확인을 방해하지 않도록 입력 폼을 직접 길게 노출하지 않는다.
- 저장 및 조회 응답은 사용자가 지연을 체감하지 않는 수준이어야 한다.
- Supabase RLS를 적용해 사용자별 데이터가 분리되어야 한다.
- 데이터 모델은 추후 다중 아기, 가족 공유, 보호자별 권한 추가가 가능해야 한다.

## 기술 요구사항

- 프론트엔드: React + Vite
- 인증: Supabase Auth 이메일/비밀번호
- 백엔드/저장소: Supabase Postgres
- 배포: Vercel
- 환경변수
  - `VITE_SUPABASE_URL`
  - `VITE_SUPABASE_PUBLISHABLE_KEY`

## 환경 및 준비 상태

- Supabase 프로젝트 생성이 필요하다.
- Supabase 이메일 인증 설정이 필요하다.
- Supabase 테이블, RLS 정책, 인덱스 구성이 필요하다.
- Vercel 프로젝트 생성 및 환경변수 설정이 필요하다.

개발 단계에서는 로컬 UI와 데이터 흐름을 먼저 구현하되, 최종 완료 기준은 Supabase Auth와 Database 연동까지 포함한다.

## 완료 기준

- 이름, 이메일, 비밀번호로 회원가입할 수 있다.
- 회원가입 후 자동 로그인된다.
- 이메일과 비밀번호로 로그인할 수 있다.
- 로그인 세션이 유지되면 재방문 시 추가 로그인 없이 진입한다.
- 최초 가입 후 아기 이름과 생년월일을 등록할 수 있다.
- 등록된 아기 이름이 홈 화면 상단에 표시된다.
- 하단 탭으로 홈, 입력, 프로필 화면을 전환할 수 있다.
- 수유, 수면, 소변, 대변 기록을 생성할 수 있다.
- 수유 기록은 0ml~300ml 범위에서 5ml 단위로 저장할 수 있다.
- 수유량 기본값은 100ml다.
- 대변 기록은 양과 색상을 저장할 수 있다.
- 생성된 기록이 최근 기록과 오늘 요약에 반영된다.
- 마지막 수유 카드에 `지금으로부터 n시간 n분 전` 형식이 표시된다.
- 마지막 수유 후 4시간이 지나면 경고 UI가 표시된다.
- 새 수유 기록을 추가하면 경고 UI가 해제된다.
- Supabase RLS 기준으로 로그인 사용자 데이터만 조회/저장된다.
- `npm run build`가 통과한다.
- Vercel 배포를 위한 환경변수 키와 Supabase 설정 절차가 문서화되어 있다.
