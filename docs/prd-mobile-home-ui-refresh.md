# Infant Time Mobile Home UI Refresh PRD

## 배경

보호자는 신생아 돌봄 중 한 손으로 모바일 화면을 열어 현재 상태를 빠르게 판단해야 한다. 기존 홈은 마지막 수유와 마지막 대변을 확인할 수 있지만, 정보가 카드 단위로 나뉘어 있어 "지금 무엇을 해야 하는지"를 즉시 판단하기 어렵다.

이번 개편은 iOS Human Interface Guidelines를 기준으로 모바일 우선 정보 구조와 시각 위계를 정리한다. 부드러운 육아 앱 톤은 유지하되, 상태 판단에 필요한 숫자와 다음 행동은 더 높은 대비로 강조한다.

## 목표

- 홈 최상단에서 현재 상태를 즉시 이해할 수 있게 한다.
- 마지막 수유 이후 경과 시간과 다음 수유까지 남은 시간을 가장 중요한 정보로 표시한다.
- 수유 텀을 progress bar 또는 countdown UI로 시각화한다.
- 수유량, 기록 시간, 대변 색/양 등 핵심 값을 숫자와 상태 중심으로 강조한다.
- 최근 기록은 타입별 아이콘과 색상을 사용해 더 조밀하게 표시한다.
- 수유/대변 기록 진입을 하단 floating CTA로 강화한다.
- iPhone Safari 기준으로 안전 영역, 하단 탭, 한 손 조작성을 고려한다.

## 디자인 기준

- 기준: iOS Human Interface Guidelines
- 원칙
  - 정보 위계를 명확히 한다.
  - 색상은 상태 전달에 일관되게 사용한다.
  - 색상만으로 타입을 구분하지 않고 아이콘과 텍스트를 함께 사용한다.
  - 숫자 정보는 크게, 보조 설명은 짧게 표시한다.
  - 주요 액션은 하단 엄지 영역에 배치하되 하단 탭과 겹치지 않게 한다.

참고 문서:

- https://developer.apple.com/design/human-interface-guidelines/color
- https://developer.apple.com/design/human-interface-guidelines/typography
- https://developer.apple.com/design/human-interface-guidelines/toolbars

## 핵심 사용자 흐름

1. 사용자는 홈 화면에 진입한다.
2. 상단에서 "마지막 수유 후 n시간 n분"을 확인한다.
3. 강조 문구와 progress bar로 다음 수유까지 남은 시간을 확인한다.
4. 마지막 수유량과 마지막 대변 상태를 한 줄 요약으로 확인한다.
5. 최근 기록 타임라인에서 최신 기록을 빠르게 스캔한다.
6. 하단 floating CTA에서 수유 또는 대변 기록 입력으로 즉시 이동한다.

## 기능 요구사항

### 1. 지금 상태 요약

- 홈 상단에 `지금 상태 요약` 영역을 배치한다.
- 마지막 수유 기록이 있으면 경과 시간을 크게 표시한다.
- 예: `마지막 수유 후 2시간 12분`
- 마지막 수유 기록이 없으면 `아직 수유 기록이 없습니다`를 표시한다.
- 다음 수유까지 남은 시간이 있으면 강조 문구로 표시한다.
- 예: `48분 후 수유 예정`
- 수유 예정 시간이 지났으면 초과 상태를 표시한다.
- 예: `수유 시간이 23분 지났어요`

### 2. 수유 텀 시각화

- 마지막 수유 시각과 저장된 수유 텀을 기준으로 progress bar를 표시한다.
- 진행률은 0% 이상 100% 이하로 clamp한다.
- 예정 시간을 초과한 경우 progress bar는 100%로 표시하고 초과 상태 색상을 적용한다.
- 수유 텀 선택값은 기존 로컬 저장 방식을 유지한다.
- 기본값은 3시간이다.

### 3. 핵심 요약 row

- 마지막 수유와 마지막 대변을 한 줄에 한 항목씩 표시한다.
- 수유 row:
  - 타입 라벨
  - 수유량 ml 강조
  - 기록 시간
  - 상대 시간
- 대변 row:
  - 타입 라벨
  - 대변 색상과 양 강조
  - 기록 시간
  - 상대 시간
- 정보가 없으면 `기록 없음` 상태를 표시한다.

### 4. 최근 기록 timeline

- 최근 기록은 compact timeline 형태로 표시한다.
- 각 기록은 타입 아이콘, 색상, 시간, 핵심 상세값을 포함한다.
- 카드 높이는 기존보다 줄이고 정보 밀도를 높인다.
- 날짜 그룹은 유지하되, 그룹별 보조 요약은 짧게 표시한다.

### 5. Floating CTA

- 홈 화면 하단에 floating CTA를 배치한다.
- 하단 탭과 safe-area를 고려해 겹치지 않게 한다.
- `수유 기록하기`를 primary 액션으로 둔다.
- `대변` 또는 `대변 기록하기`를 secondary 액션으로 둔다.
- 버튼 클릭 시 기존 입력 탭으로 이동하고 해당 타입이 선택되어야 한다.

## 시각 디자인 가이드

- 배경: 기존 부드러운 pastel gradient 유지
- 상태 카드: 밝은 surface에 높은 텍스트 대비 적용
- 주요 숫자: 32px 이상 권장
- 보조 텍스트: 13px~15px
- 타입 색상
  - 수유: blue
  - 대변: warm brown/yellow
  - 소변: yellow
  - 수면: indigo
- radius: 기존 앱과 맞춰 8px 중심
- 그림자: 약하게 사용하고 정보 카드 내부 대비를 우선한다.

## 수용 기준

- iPhone Safari 375px 폭에서 가로 스크롤이 없어야 한다.
- 홈 상단에서 마지막 수유 경과 시간과 다음 수유 남은 시간이 보여야 한다.
- progress bar가 수유 텀 진행률을 반영해야 한다.
- 수유 예정 시간이 지나면 초과 상태 문구가 보여야 한다.
- 마지막 수유/대변은 한 줄에 한 항목씩 보여야 한다.
- 최근 기록은 아이콘과 색상으로 타입이 구분되어야 한다.
- floating CTA에서 수유와 대변 입력 화면으로 바로 진입할 수 있어야 한다.
- `npm run build`가 통과해야 한다.
